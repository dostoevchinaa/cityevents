Выполненные работы

### 1. Выбор CMS и установка
- Выбрана CMS WordPress
- Сайт развёрнут на бесплатном хостинге Beget (домен: http://k96147la.beget.tech)

### 2. Создание структуры данных
- Зарегистрирован кастомный тип записи `event` (События)
- Добавлены мета-поля: `place`, `start_at`, `end_at`, `tags`, `capacity`, `status`, `popularity`, `change_number`
- Все поля доступны в REST API и в админке WordPress

### 3. REST API эндпоинты
Созданы следующие эндпоинты GET список всех событий, GET получение одного события, POST, PUT, DELETE

POST, PUT, DELETE методы не удалось протестировать на бесплатном хостинге из-за ограничений (блокировка не-GET запросов и авторизации).

### 4. Реализованный функционал

Расчет popularity
Реализована автоматическая формула расчета популярности события:

days_to_start = start_date - today_date
raw = 3 + (capacity / 1000) - (days_to_start / 10) + количество_тегов
popularity = raw, приведённый к диапазону 1..5

При популярности = 1 создание события блокируется с ошибкой 400.

Сортировка (GET /events)
По умолчанию события сортируются:
- Сначала по дате начала (ближайшие выше)
- При равных датах — по убыванию popularity
- При равенстве — по возрастанию id

Валидация данных
- Проверка обязательных полей (title, place, start_at, end_at, capacity)
- Проверка capacity (1-5000)
- Проверка количества тегов (≤5)
- Проверка дат (start_at < end_at)
- Запрет изменения прошедших дат
- Запрет возврата отменённых событий в published

Авторизация
Реализована Basic Auth через фильтры WordPress, но на бесплатном хостинге Beget авторизация не работает из-за ограничений сервера.
